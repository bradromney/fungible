# Architecture Decision Records (ADRs)

Each ADR captures one significant decision, the context, the choice, and the
trade-offs we knowingly accepted. They are immutable once accepted — if we
change our mind, we add a new ADR that supersedes the old one rather than
editing history. This gives us a durable record of *why* the system looks the
way it does as the project evolves.

Format: lightweight [MADR](https://adr.github.io/madr/)-style.
